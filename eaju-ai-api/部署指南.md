# 智蚁 AI API - 测试环境部署指南

## 📋 部署前准备

### 环境要求
- PostgreSQL 12+
- Redis 5+
- JDK 8+
- Node.js 16+（前端）

1. 获取各 AI 平台的 API Key：
   - DeepSeek: https://platform.deepseek.com/
   - 通义千问：https://dashscope.console.aliyun.com/
   - Kimi: https://platform.moonshot.cn/
   - Google Gemini: https://makersuite.google.com/
   - 百度千帆：https://console.bce.baidu.com/qianfan/

2. 准备 JWT 密钥（至少 32 字节）
3. 准备 Redis 连接信息
4. 准备数据库连接信息

## 🗄️ 数据库部署

### 1. 创建数据库

```bash
# 连接到 PostgreSQL
psql -U postgres -h 你的数据库主机

# 创建数据库
CREATE DATABASE eaju_ai
    WITH
    OWNER = postgres
    ENCODING = 'UTF8'
    CONNECTION LIMIT = -1;

\q
```

### 2. 执行建表脚本

```bash
# 方式一：使用完整部署脚本（推荐）
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/deploy.sql

# 方式二：使用分步脚本
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/01-chat-turn-table.sql
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/02-llm-provider-config.sql
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/03-chat-conversation.sql
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/04-api-key.sql
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/06-chat-conversation-last-model.sql
psql -U postgres -h 你的数据库主机 -d eaju_ai -f db/08-api-key-welcome-config.sql
```

### 3. 更新 API Key

```bash
# 连接到数据库
psql -U postgres -h 你的数据库主机 -d eaju_ai

-- 更新各平台 API Key
UPDATE llm_provider_config SET api_key = '你的 DeepSeek 密钥' WHERE code = 'DEEPSEEK';
UPDATE llm_provider_config SET api_key = '你的通义千问密钥' WHERE code = 'QWEN';
UPDATE llm_provider_config SET api_key = '你的 Kimi 密钥' WHERE code = 'KIMI';
UPDATE llm_provider_config SET api_key = '你的 Gemini 密钥' WHERE code = 'GEMINI';
UPDATE llm_provider_config SET api_key = '你的千帆密钥' WHERE code = 'QIANFAN';
UPDATE llm_provider_config SET api_key = '你的 MiniMax 密钥' WHERE code = 'MINIMAX';

-- 验证更新
SELECT code, display_name, LEFT(api_key, 10) as api_key_preview FROM llm_provider_config;

\q
```

## 🔧 后端部署

### 1. 配置环境变量

创建 `.env` 文件或配置环境变量：

```bash
# 数据库配置
SPRING_DATASOURCE_URL=jdbc:postgresql://你的数据库主机：5432/eaju_ai
SPRING_DATASOURCE_USERNAME=postgres
SPRING_DATASOURCE_PASSWORD=你的数据库密码

# Redis 配置
REDIS_HOST=你的 Redis 主机
REDIS_PORT=6379
REDIS_PASSWORD=你的 Redis 密码（如有）

# JWT 配置（生产环境务必使用强密钥）
APP_JWT_SECRET=你的 JWT 密钥（至少 32 字节，建议使用随机字符串）

# 服务器端口
SERVER_PORT=8080

#  HikariCP 连接池配置（可选）
HIKARI_MAX_POOL_SIZE=20
HIKARI_MIN_IDLE=5

# HTTP 客户端配置（可选）
HTTP_MAX_TOTAL=50
HTTP_MAX_PER_ROUTE=10
```

### 2. 编译打包

```bash
cd eaju-ai-api

# Maven 编译
mvn clean package -DskipTests

# 或使用 Gradle（如果项目使用 Gradle）
# ./gradlew clean build -x test
```

### 3. 启动服务

```bash
# 方式一：直接运行 jar
java -jar target/eaju-ai-api-1.0.0-SNAPSHOT.jar

# 方式二：使用环境变量
export SPRING_DATASOURCE_URL=jdbc:postgresql://主机：5432/eaju_ai
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=密码
export REDIS_HOST=redis 主机
java -jar target/eaju-ai-api-1.0.0-SNAPSHOT.jar

# 方式三：后台运行
nohup java -jar target/eaju-ai-api-1.0.0-SNAPSHOT.jar > app.log 2>&1 &
```

### 4. 验证服务

```bash
# 检查服务状态
curl http://localhost:8080/actuator/health

# 访问 Swagger UI
open http://localhost:8080/swagger-ui.html
```

## 💻 前端部署

### 1. 安装依赖

```bash
cd eaju-ai-web

# 安装 npm 依赖
npm install
```

### 2. 配置环境变量

创建 `.env` 文件（基于 `.env.example`）：

```bash
# API 基础地址
VITE_API_BASE_URL=http://你的后端地址：8080

# 其他配置（如有）
```

### 3. 编译打包

```bash
# 生产环境编译
npm run build

# 编译产物在 dist/ 目录
```

### 4. 部署到 Web 服务器

#### 方式一：Nginx 部署

```nginx
server {
    listen 80;
    server_name 你的域名;
    
    root /path/to/eaju-ai-web/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 代理（可选，如需反向代理）
    location /api/ {
        proxy_pass http://后端地址：8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

#### 方式二：Docker 部署

```dockerfile
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```bash
# 构建镜像
docker build -t eaju-ai-web .

# 运行容器
docker run -d -p 80:80 eaju-ai-web
```

## ⚠️ 常见问题

### 1. 数据库连接失败

```bash
# 检查 PostgreSQL 服务
systemctl status postgresql

# 检查连接字符串
echo $SPRING_DATASOURCE_URL

# 测试数据库连接
psql -U postgres -h 主机 -d eaju_ai
```

### 2. Redis 连接失败

```bash
# 检查 Redis 服务
systemctl status redis

# 测试 Redis 连接
redis-cli -h 主机 ping
```

### 3. API Key 无效

```bash
# 检查 API Key 配置
psql -U postgres -h 主机 -d eaju_ai -c "SELECT code, LEFT(api_key, 10) FROM llm_provider_config;"

# 更新 API Key
UPDATE llm_provider_config SET api_key = '新密钥' WHERE code = 'DEEPSEEK';
```

### 4. 前端无法连接后端

```bash
# 检查 API 地址配置
cat .env | grep API_BASE_URL

# 检查后端服务
curl http://后端地址：8080/actuator/health

# 检查 CORS 配置
# 确保后端允许前端域名访问
```

## 🔐 安全建议

1. **数据库密码**：使用强密码，定期更换
2. **JWT 密钥**：使用至少 32 字节的随机字符串
3. **API Key**：妥善保管，不要提交到代码仓库
4. **HTTPS**：生产环境务必使用 HTTPS
5. **防火墙**：限制数据库和 Redis 的访问 IP
6. **备份**：定期备份数据库
7. **日志**：定期清理日志，避免磁盘占满
