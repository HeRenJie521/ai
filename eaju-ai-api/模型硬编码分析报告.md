## 三、推荐配置化的项目（可选）

以下硬编码如需要更灵活的控制，可考虑配置化：

### 3.1 工具调用最大轮次

**当前状态**：硬编码 `MAX_ROUNDS = 5`

**配置化方案**：
1. 在 `llm_model` 表增加字段 `max_tool_call_rounds INTEGER DEFAULT 5`
2. 在 `ToolCallOrchestrator` 中按模型读取配置
3. 允许不同模型有不同的最大轮次（如复杂任务模型可设置 10 轮）

**SQL 示例**：
```sql
ALTER TABLE llm_model ADD COLUMN max_tool_call_rounds INTEGER DEFAULT 5;
COMMENT ON COLUMN llm_model.max_tool_call_rounds IS '工具调用最大轮次，防止无限循环';
```

### 3.2 流式超时时间

**当前状态**：硬编码 `STREAM_TIMEOUT_MS = 30 * 60 * 1000L`

**配置化方案**：
1. 在 `application.yml` 中增加配置项 `app.chat.stream-timeout-ms`
2. 默认值 30 分钟，可通过环境变量覆盖

**YAML 示例**：
```yaml
app:
  chat:
    stream-timeout-ms: ${APP_CHAT_STREAM_TIMEOUT_MS:1800000}  # 默认 30 分钟
```

### 3.3 会话标题最大长度

**当前状态**：硬编码 `TITLE_MAX = 48`

**配置化方案**：
1. 在 `application.yml` 中增加配置项 `app.chat.title-max-length`
2. 默认值 48，可通过环境变量覆盖

**YAML 示例**：
```yaml
app:
  chat:
    title-max-length: ${APP_CHAT_TITLE_MAX_LENGTH:48}
```

### 3.4 文件上传最大大小

**当前状态**：硬编码 `MAX_FILE_SIZE = 5 * 1024 * 1024L`

**配置化方案**：
1. 在 `application.yml` 中增加配置项 `app.upload.max-file-size`
2. 默认值 5MB，可通过环境变量覆盖

**YAML 示例**：
```yaml
app:
  upload:
    max-file-size: ${APP_UPLOAD_MAX_FILE_SIZE:5242880}  # 默认 5MB
```
