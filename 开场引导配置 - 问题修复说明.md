# 开场引导配置功能 - 问题修复

## 修复的问题

### 1. ✅ 移除图标
- **问题**：开场白上方有图标显示
- **修复**：移除了 welcome-icon 元素及其样式
- **效果**：开场白简洁展示，无图标

### 2. ✅ 点击推荐问题正常发送
- **问题**：点击推荐问题后页面无反应
- **原因**：handleSuggestionClick 函数中 welcomeConfig.value = null 可能导致响应式更新问题
- **修复**：移除 `welcomeConfig.value = null` 代码，保持开场白显示
- **效果**：点击推荐问题后，问题自动发送并显示 AI 回复

### 3. ✅ 开场白左对齐
- **问题**：开场白居中显示，与 AI 回复样式不一致
- **修复**：
  - `.welcome-section` 改为 `justify-content: flex-start`
  - `.welcome-bubble` 背景色改为 `#f4f4f5`（与 AI 消息气泡一致）
  - `.welcome-bubble` 添加 `border-bottom-left-radius: 4px`（AI 消息样式）
  - `.suggestion-btn` 改为 `justify-content: flex-start`
- **效果**：开场白和推荐问题左对齐，与 AI 回复样式一致

### 4. ✅ 开场白永远显示在第一个位置
- **问题**：
  - 点击推荐问题后开场白消失
  - 有会话记录后开场白不显示
- **修复**：
  - 模板条件从 `v-if="messages.length === 0 && welcomeConfig..."` 改为 `v-if="welcomeConfig && welcomeConfig.welcomeText"`
  - 页面加载时始终调用 `getWelcomeConfig(iid)`，不再判断 messages 长度
  - 移除 `welcomeConfig.value = null` 代码
- **效果**：
  - 开场白永远显示在消息列表第一个位置
  - 点击推荐问题后开场白不消失
  - 有历史会话时开场白仍然显示

## 技术细节

### 关键代码变更

**模板条件：**
```vue
<!-- 之前 -->
<div v-if="messages.length === 0 && welcomeConfig && welcomeConfig.welcomeText">

<!-- 之后 -->
<div v-if="welcomeConfig && welcomeConfig.welcomeText">
```

**加载逻辑：**
```typescript
// 之前：仅当没有消息时加载
if (messages.value.length === 0) {
  welcomeConfig.value = await getWelcomeConfig(iid)
}

// 之后：始终加载
welcomeConfig.value = await getWelcomeConfig(iid)
```

**点击处理：**
```typescript
// 之前：点击后隐藏
welcomeConfig.value = null
void sendMessage(question)

// 之后：保持显示
void sendMessage(question)
```

### 样式变更

```css
/* 之前：居中、白色背景、有图标 */
.welcome-section { justify-content: center; }
.welcome-bubble { background: #fff; box-shadow: ...; }

/* 之后：左对齐、灰色背景、无图标 */
.welcome-section { justify-content: flex-start; }
.welcome-bubble { background: #f4f4f5; border-bottom-left-radius: 4px; }
```

## 注意事项

1. **性能考虑**
   - 开场白配置始终加载，但数据量小，不影响性能
   - 推荐问题按钮有防重复点击机制

2. **兼容性**
   - 保持与现有聊天逻辑兼容
   - 不影响 API Key 类型集成

3. **扩展性**
   - 开场白配置数据结构保持不变
   - 未来可支持更复杂的展示形式
