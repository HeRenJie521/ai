# 开场引导配置功能实现文档

## 功能概述

为"集成管理 -> 嵌入网站"类型新增开场引导配置功能，当用户打开聊天页面且当前没有任何对话记录时，展示开场白和推荐问题。

## 数据结构设计

### 数据库变更

在 `api_key` 表中新增字段（仅对 `type=2` 嵌入网站类型生效）：

```sql
-- 数据库迁移脚本：db/08-api-key-welcome-config.sql
ALTER TABLE api_key
ADD COLUMN welcome_text TEXT,        -- 开场白文本
ADD COLUMN suggestions JSONB;        -- 推荐问题 JSON 数组
```

**字段说明：**
- `welcome_text`: TEXT 类型，开场白内容
- `suggestions`: JSONB 类型，存储字符串数组，例如 `["问题 1", "问题 2", "问题 3"]`

### 实体类更新

**ApiKeyEntity.java** 新增字段：
```java
@Column(name = "welcome_text", columnDefinition = "TEXT")
private String welcomeText;

@Column(name = "suggestions", columnDefinition = "JSONB")
private String suggestions;
```

## 后端实现

### 1. 获取开场引导配置接口

**接口地址：** `GET /api/chat/welcome?id={integrationId}`

**请求参数：**
- `id`: 集成 ID（api_key.id）

**响应示例：**
```json
{
  "welcomeText": "你好，我是 AI 助手，有什么可以帮助您的吗？",
  "suggestions": ["产品有哪些功能？", "如何收费？", "联系方式是什么？"]
}
```

**实现类：** `WelcomeConfigController.java`

### 2. 更新集成配置接口

**接口地址：** `PATCH /api/admin/api-keys/{id}`

**请求体新增字段：**
```json
{
  "name": "集成名称",
  "enabled": true,
  "welcomeText": "开场白文本",
  "suggestions": "[\"问题 1\", \"问题 2\"]"
}
```

**修改文件：**
- `ApiKeyPatchRequestDto.java` - 新增 welcomeText、suggestions 字段
- `ApiKeyResponseDto.java` - 新增 welcomeText、suggestions 字段
- `ApiKeyService.java` - update 方法支持新字段
- `AdminApiKeyController.java` - patch 方法传递新字段

## 前端实现

### 1. 管理端配置页面（ApiKeysView.vue）

在编辑嵌入网站集成时，新增两个配置项：

**开场白文本（可选）**
- 多行文本输入框
- 提示：用户打开聊天页面且无对话记录时显示

**推荐问题（可选，JSON 数组格式）**
- 多行文本输入框
- 格式示例：`["问题 1", "问题 2", "问题 3"]`
- 前端验证 JSON 格式
- 提示：格式：["问题 1", "问题 2"]，用户点击后自动发送

**仅当集成类型为"嵌入网站"（type=2）时显示以上配置项。**

### 2. 嵌入聊天页面（EmbedView.vue）

#### 状态变量
```typescript
const welcomeConfig = ref<WelcomeConfig | null>(null)
const welcomeLoaded = ref(false)
const suggestionsSending = ref(false)
```

#### 加载逻辑
1. 页面初始化时，如果当前没有对话记录，调用 `getWelcomeConfig(iid)` 获取配置
2. 新建会话时，重新加载开场引导配置

#### 展示逻辑
- 当 `messages.length === 0` 且存在 `welcomeConfig.welcomeText` 时，显示开场引导区域
- 否则显示默认空状态提示

#### 推荐问题交互
1. 点击推荐问题按钮
2. 检查是否正在发送消息（防重复点击）
3. 隐藏开场引导区域
4. 自动发送该问题并触发 AI 回复
5. 发送过程中显示加载动画

### 3. API 接口封装

**新文件：** `src/api/welcome.ts`

```typescript
export interface WelcomeConfig {
  welcomeText: string | null
  suggestions: string[] | null
}

export async function getWelcomeConfig(integrationId: number): Promise<WelcomeConfig>
```

**更新文件：** `src/api/adminApiKeys.ts`
- `ApiKeyRow` 接口新增 `welcomeText`、`suggestions` 字段
- `adminPatchApiKey` 方法支持新参数

## 交互细节

### 1. 显示条件
- ✅ 用户打开聊天页面
- ✅ 当前没有任何对话记录（`messages.length === 0`）
- ✅ 已配置开场白文本（`welcomeText` 不为空）

### 2. 点击推荐问题
- ✅ 防止重复点击（sending/suggestionsSending 检查）
- ✅ 点击后立即隐藏开场引导区域
- ✅ 按钮显示加载动画
- ✅ 自动发送问题并触发 AI 回复

### 3. 新建会话
- ✅ 点击"新建会话"时重新加载开场引导
- ✅ 清空消息列表后自动显示

### 4. 无配置情况
- ✅ 如果没有配置开场白，显示默认空状态提示
- ✅ 如果推荐问题为空，不显示推荐问题区域

## 使用示例

### 配置开场引导

1. 登录管理后台
2. 进入"集成管理"页面
3. 找到类型为"嵌入网站"的集成
4. 点击"编辑"
5. 填写：
   - 开场白文本：`你好，我是 AI 助手，有什么可以帮助您的吗？`
   - 推荐问题：`["产品有哪些功能？", "如何收费？", "技术支持联系方式"]`
6. 点击"保存"

### 前端效果

用户打开嵌入的聊天页面时：
1. 如果没有任何对话记录，显示开场白和 3 个推荐问题按钮
2. 用户点击任意推荐问题
3. 问题自动发送到聊天窗口
4. AI 开始回复

## 扩展预留

`suggestions` 字段后续可以扩展为对象数组：

```json
{
  "suggestions": [
    {
      "title": "产品功能",
      "content": "产品有哪些功能？"
    },
    {
      "title": "价格咨询",
      "content": "如何收费？"
    }
  ]
}
```

当前版本仅支持简单的字符串数组，未来可以支持标题 + 内容的展示形式。

## 注意事项

1. **数据库兼容性**：JSONB 是 PostgreSQL 特有类型，如使用其他数据库需调整
2. **字段长度**：welcome_text 使用 TEXT 类型，支持长文本
3. **安全性**：suggestions JSON 需要后端验证格式，避免恶意数据
4. **性能**：开场引导配置会缓存在前端，避免重复请求
5. **兼容性**：仅对 type=2（WEB_EMBED）类型生效，不影响 API_KEY 类型
