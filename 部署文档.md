# 部署文档

## 服务器配置要求

### 最低配置
- **CPU**: 2核
- **内存**: 4GB
- **磁盘**: 40GB SSD
- **操作系统**: Ubuntu 22.04 LTS / CentOS 8+

### 推荐配置
- **CPU**: 4核
- **内存**: 8GB
- **磁盘**: 80GB SSD
- **操作系统**: Ubuntu 22.04 LTS

## 环境依赖

| 组件 | 版本要求 |
|------|---------|
| JDK | 1.8 (Java 8) |
| Node.js | 18+ |
| PostgreSQL | 14+（本地开发使用 17.9，测试/生产环境推荐 15+） |
| Nginx | 1.20+（可选，用于反向代理） |

## 一、后端部署（eaju-ai-api）

### 1. 数据库准备

```sql
-- 创建数据库
CREATE DATABASE eaju_ai;

-- 创建用户（可选）
CREATE USER eaju_ai_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE eaju_ai TO eaju_ai_user;

-- 执行迁移脚本
\c eaju_ai
\i src/main/resources/db/V1__initial_schema.sql
\i src/main/resources/db/V2__alter_secret_prefix_length.sql
```

### 2. 配置修改

编辑 `src/main/resources/application.properties`（或 `application.yml`）：

```properties
# 数据库配置
spring.datasource.url=jdbc:postgresql://localhost:5432/eaju_ai
spring.datasource.username=eaju_ai_user
spring.datasource.password=your_password

# JWT 配置
jwt.secret=your_jwt_secret_key_at_least_256_bits_long
jwt.expiration=86400

# LLM 服务配置（根据实际情况配置）
llm.providers.qwen.api-key=your_qwen_api_key
llm.providers.qwen.base-url=https://dashscope.aliyuncs.com

# 服务器端口
server.port=8080
```

### 3. 编译打包

```bash
cd eaju-ai-api
# 如果使用 Maven Wrapper
./mvnw clean package -DskipTests

# 或者直接使用 mvn
mvn clean package -DskipTests
```

编译成功后，产物位于：`target/eaju-ai-api-*.jar`

> **注意**：确保编译环境与运行环境的 JDK 版本一致（JDK 1.8）。

### 4. 启动服务

```bash
# 方式1：直接启动
java -jar target/eaju-ai-api-*.jar --spring.profiles.active=prod

# 方式2：后台启动
nohup java -jar target/eaju-ai-api-*.jar \
  --spring.profiles.active=prod \
  --server.port=8080 \
  > app.log 2>&1 &

# 方式3：使用 systemd（推荐）
cat > /etc/systemd/system/eaju-ai.service << 'EOF'
[Unit]
Description=Eaju AI API Service
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/eaju-ai
ExecStart=/usr/bin/java -jar eaju-ai-api.jar --spring.profiles.active=prod
Restart=on-failure
RestartSec=10
Environment=JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable eaju-ai
sudo systemctl start eaju-ai
sudo systemctl status eaju-ai
```

### 5. 验证

```bash
curl http://localhost:8080/api/health
# 预期返回：{"status":"UP"}
```

## 二、前端部署（eaju-ai-web）

### 1. 安装依赖

```bash
cd eaju-ai-web
npm install
```

### 2. 环境变量配置

创建 `.env.production` 文件（如果后端不在同一域名）：

```env
# 后端 API 地址
VITE_API_BASE_URL=http://your-server-ip:8080
```

> **注意**：如果前后端部署在同一域名（通过 Nginx 反向代理），则不需要配置此项。

### 3. 编译构建

```bash
npm run build
```

该命令会执行：
1. `vue-tsc -b` — TypeScript 类型检查
2. `vite build` — 生产环境打包

构建成功后，产物位于：`dist/` 目录

> **常见问题**：如果 TypeScript 检查报错但你想跳过，可以运行 `npx vite build` 直接打包。

### 4. 本地预览（可选）

```bash
# 使用 Vite 内置预览服务
npm run preview

# 默认端口：4173
# 访问：http://localhost:4173
```

### 5. 修改端口（如需要）

#### 方式1：修改 vite.config.ts

编辑 `vite.config.ts`，添加 `server` 配置：

```typescript
export default defineConfig({
  plugins: [vue()],
  server: {
    port: 3000,        // 开发环境端口
    strictPort: true   // 端口被占用时直接报错，而不是自动切换
  },
  preview: {
    port: 3000,        // 预览环境端口
    strictPort: true
  }
})
```

#### 方式2：通过命令行参数指定

```bash
# 开发环境
npm run dev -- --port 3000

# 预览环境
npm run preview -- --port 3000
```

#### 方式3：使用环境变量

```bash
# Linux/Mac
PORT=3000 npm run dev

# Windows (PowerShell)
$env:PORT=3000; npm run dev
```

### 6. 部署到 Web 服务器

#### 方式1：使用 Nginx（推荐）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/eaju-ai-web;
    index index.html;

    # 前端静态资源
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持（如需流式输出）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

```bash
# 部署静态资源
sudo cp -r dist/* /var/www/eaju-ai-web/
sudo chown -R www-data:www-data /var/www/eaju-ai-web

# 重启 Nginx
sudo systemctl restart nginx
```

#### 方式2：使用 Node.js 静态服务器（临时预览）

```bash
# 安装 serve
npm install -g serve

# 启动服务（默认端口 3000）
serve -s dist -l 3000

# 指定端口
serve -s dist -l 8080
```

#### 方式3：使用 Python 简单服务器（临时预览）

```bash
# Python 3
cd dist
python3 -m http.server 8080

# 访问：http://localhost:8080
```

> **注意**：方式2和方式3仅适用于临时预览，生产环境请使用 Nginx。

## 三、完整部署流程

### 1. 部署顺序

```
1. PostgreSQL 数据库
2. 后端 API 服务（eaju-ai-api）
3. 前端静态资源（eaju-ai-web）
4. Nginx 反向代理（可选）
```

### 2. 一键部署脚本（示例）

```bash
#!/bin/bash
set -e

echo "=== 开始部署 ==="

# 1. 后端
echo ">>> 编译后端..."
cd eaju-ai-api && mvn clean package -DskipTests
sudo cp target/*.jar /opt/eaju-ai/eaju-ai-api.jar
sudo systemctl restart eaju-ai

# 2. 前端
echo ">>> 编译前端..."
cd ../eaju-ai-web && npm install && npm run build
sudo cp -r dist/* /var/www/eaju-ai-web/
sudo systemctl restart nginx

echo "=== 部署完成 ==="
```

### 3. 健康检查

```bash
# 后端
curl http://localhost:8080/api/health

# 前端
curl http://localhost/

# 数据库连接
curl http://localhost:8080/api/admin/health
```

## 五、常见问题

### 1. 端口被占用

```bash
# 查看占用端口的进程
sudo lsof -i :8080
sudo lsof -i :80

# 终止进程
sudo kill -9 <PID>
```

### 2. 数据库连接失败

```bash
# 检查 PostgreSQL 状态
sudo systemctl status postgresql

# 检查数据库配置
psql -U postgres -c "\l" | grep eaju_ai
```

### 3. 前端 404

确保 Nginx 配置了 `try_files $uri $uri/ /index.html;`

### 4. CORS 问题

如果前后端分离部署，确保后端配置了 CORS：

```java
// SecurityConfig.java 中配置
.cors(cors -> cors.configurationSource(request -> {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(List.of("http://your-frontend-domain"));
    config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "OPTIONS"));
    config.setAllowedHeaders(List.of("*"));
    config.setAllowCredentials(true);
    return config;
}))
```

## 六、性能优化建议

1. **启用数据库连接池**
   ```properties
   spring.datasource.hikari.maximum-pool-size=20
   spring.datasource.hikari.minimum-idle=5
   ```

2. **启用 HTTP/2**
   ```nginx
   listen 443 ssl http2;
   ```

3. **配置 CDN**（静态资源）

4. **开启 Gzip/Brotli 压缩**

5. **配置 HTTPS**
   ```bash
   sudo certbot --nginx -d your-domain.com
   ```

6. **监控与告警**
   - 使用 Prometheus + Grafana 监控服务状态
   - 配置日志聚合（ELK/Loki）
