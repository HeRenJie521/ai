# 开场引导配置 - 最终修复

## 修复的问题

### 1. ✅ 开场白和推荐问题要有背景框包住

**问题**：开场白和推荐问题没有像 AI 回复那样的背景气泡框

**修复方案**：
- 将开场白容器从 `welcome-section` + `welcome-bubble` 改为使用 AI 消息的样式类
- 直接使用 `msg msg--ai` 和 `bubble bubble--ai` 类
- 移除多余的嵌套 div

**修改前**：
```vue
<div class="welcome-section">
  <div class="welcome-bubble">
    <div class="welcome-content">
      <p class="welcome-text">...</p>
      <div class="suggestions-list">...</div>
    </div>
  </div>
</div>
```

**修改后**：
```vue
<div class="msg msg--ai">
  <div class="bubble bubble--ai">
    <p class="welcome-text">...</p>
    <div class="suggestions-list">...</div>
  </div>
</div>
```

### 2. ✅ 点击推荐问题后没有发送

**问题**：点击推荐问题后，页面没有任何响应，问题没有发送

**原因分析**：
1. `@click.stop` 阻止了事件传播
2. 可能存在状态检查阻止了发送

**修复方案**：
1. 将 `@click.stop` 改为 `@click`
2. 简化状态检查逻辑
3. 添加 provider 检查，确保有选中的模型

**修改前**：
```vue
@click.stop="handleSuggestionClick(suggestion)"
```

```typescript
function handleSuggestionClick(question: string) {
  if (suggestionsSending.value || sending.value) return
  suggestionsSending.value = true
  void sendMessage(question).finally(() => {
    suggestionsSending.value = false
  })
}
```

**修改后**：
```vue
@click="handleSuggestionClick(suggestion)"
```

```typescript
function handleSuggestionClick(question: string) {
  if (suggestionsSending.value || sending.value) {
    return
  }
  suggestionsSending.value = true
  void sendMessage(question).finally(() => {
    suggestionsSending.value = false
  })
}

// sendMessage 函数中添加
if (!selectedProvider.value) {
  console.error('No provider selected')
  return
}
```

## 修改的文件

### EmbedView.vue

#### 1. 模板部分
```vue
<!-- 开场引导区域（永远显示在第一个位置） -->
<div v-if="welcomeConfig && welcomeConfig.welcomeText" class="msg msg--ai">
  <div class="bubble bubble--ai">
    <p class="welcome-text">{{ welcomeConfig.welcomeText }}</p>
    <!-- 推荐问题列表 -->
    <div v-if="welcomeConfig.suggestions && welcomeConfig.suggestions.length > 0" class="suggestions-list">
      <button
        v-for="(suggestion, index) in welcomeConfig.suggestions"
        :key="index"
        :class="['suggestion-btn', { 'suggestion-btn--loading': suggestionsSending }]"
        :disabled="suggestionsSending || sending"
        @click="handleSuggestionClick(suggestion)"
      >
        <span v-if="!suggestionsSending">{{ suggestion }}</span>
        <span v-else class="suggestion-loading">
          <span class="dot" /><span class="dot" /><span class="dot" />
        </span>
      </button>
    </div>
  </div>
</div>
```

#### 2. 脚本部分
- `handleSuggestionClick`: 移除 `.stop` 修饰符的影响
- `sendMessage`: 添加 provider 检查

#### 3. 样式部分
移除了自定义的 `.welcome-section` 和 `.welcome-bubble` 样式，直接使用 AI 消息的样式类。

保留的样式：
- `.welcome-text`: 文本样式
- `.suggestions-list`: 列表布局
- `.suggestion-btn`: 按钮样式
- `.suggestion-loading`: 加载动画

## 调试建议

如果点击推荐问题仍然没有反应，检查：

1. **浏览器控制台**
   - 是否有 "No provider selected" 错误
   - 是否有网络请求错误
   - 请求 URL 是否正确（应该是 `/api/chat`）

2. **状态检查**
   - `selectedProvider.value` 是否有值
   - `authStore.token` 是否存在
   - `sending.value` 和 `suggestionsSending.value` 状态

3. **网络请求**
   - 请求是否发出
   - 响应状态码
   - 响应内容格式
